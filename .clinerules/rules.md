# Project Rules

* Always minimize token usage
* Never scan the entire repository unless explicitly required
* Only read files relevant to the current task
* Avoid repeating explanations
* Keep responses concise
* Prefer editing existing files instead of creating duplicates
* Summarize completed work briefly
* Use short plans
* Avoid unnecessary tool calls
* Keep context lightweight

When context becomes large:

* summarize progress
* update memory bank
* recommend using /smol
* recommend using /newtask

Always read the memory bank before starting work.
